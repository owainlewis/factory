# Product Experiment Policy

Use this policy for active tools, prototypes, and experiments.

Examples:

- `neo`
- `cortex`
- `push`

## Allowed

- Fix docs that are clearly stale.
- Add smoke tests for documented behavior.
- Fix failing CI when the cause is clear.
- Improve examples.
- Open issues for missing features or unclear behavior.
- Prepare release checklists.

## Human Review Required

Human review is required before:

- changing product scope
- removing features
- changing CLI contracts
- adding major dependencies
- changing storage formats
- cutting releases

## Quality Bar

A product experiment should make clear:

- what works now
- what does not work yet
- how to run it
- how to test it
- what the next useful improvement is
