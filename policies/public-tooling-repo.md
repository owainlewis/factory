# Public Tooling Repo Policy

Use this policy for public developer tooling and agent tooling repos.

## Allowed

- Improve docs.
- Add examples.
- Fix tests and lint.
- Improve CI when the fix is clear.
- Open issues for missing usage docs or unclear behavior.

## Human Review Required

Human review is required before:

- changing public APIs
- changing CLI behavior
- adding major dependencies
- changing install commands
- changing package names
- cutting releases

## Quality Bar

Public tooling should be easy to evaluate quickly.

The repo should show:

- install command
- minimal example
- test command
- supported platforms
- current limitations
