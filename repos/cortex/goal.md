# Cortex Goal

## Mission

Build a fast, minimalist terminal code editor in Rust.

## Current Focus

Make the editor reliable, well documented, and easy to test as a small technical proof project.

## Factory Should Improve

- README accuracy.
- Test coverage for core editing behavior.
- CI health.
- Issue quality.
- Release readiness.
- Known limitations.

## Factory Should Avoid

- Expanding scope too early.
- Changing editor behavior without review.
- Adding plugins, LSP, or AI features before the basic editor loop is solid.

## Definition Of Better

The repo is better when:

- the editor can be built and smoke tested
- core behavior has tests
- limitations are clear
- issues describe the next useful improvements
- small PRs keep the project moving
