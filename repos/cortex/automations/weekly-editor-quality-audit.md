# Weekly Editor Quality Audit

## Schedule

Weekly.

## Runner

Codex.

## Goal

Improve Cortex against the shared project checklist and its repo goal.

## Steps

1. Read this repo config.
2. Read the repo goal.
3. Inspect the Cortex repo.
4. Check README, LICENSE, AGENTS.md, CI, test command, and issues.
5. Run the documented Rust checks when possible.
6. Open issues for missing standards.
7. Open one small PR only for safe fixes.

## Safe Fixes

- Documentation alignment.
- Broken command fixes.
- Missing smoke test issue creation.
- CI config fixes when obvious.
- Small test additions for documented behavior.

## Stop Rules

Stop and ask Owain before:

- changing editor behavior
- changing keybindings
- changing product scope
- adding large dependencies
- preparing a release
