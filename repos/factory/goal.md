# Factory Goal

## Mission

Code Factory should become the simple control plane for improving important GitHub repos every day.

## Current Focus

Get the architecture right before writing automation code.

## Factory Should Improve

- Repo configs.
- Shared standards.
- Standard labels.
- Project board requirements.
- Automation prompts.
- Reports that make the next action obvious.

## Factory Should Avoid

- Building a complex platform too early.
- Adding every repo at once.
- Creating vague docs.
- Running destructive changes.

## Definition Of Better

Factory is better when a fresh agent can read a repo folder and know:

- what the repo is for
- what standards apply
- what work is allowed
- what automation should run
- when to ask Owain
