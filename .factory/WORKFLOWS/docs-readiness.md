# Docs Readiness

## Goal

Keep Factory docs accurate, clear, and useful for someone trying to run the system.

## Plan Mode

In plan mode:

1. Read README and docs.
2. Compare docs against current code behavior.
3. Identify stale names, missing commands, and unsupported claims.
4. Report one smallest safe docs fix.
5. Do not edit files.

## Execute Mode

In execute mode:

1. Make one focused docs fix.
2. Keep claims grounded in current code.
3. Run relevant checks.
4. Open a draft pull request.

## Stop Rules

- Do not invent roadmap promises.
- Do not make public claims without evidence.
- Do not rewrite broad docs when one focused fix is enough.
